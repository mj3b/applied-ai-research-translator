# v1.1 Reviewer Brief

This brief gives reviewers a compressed path through the v1.1 release of Applied AI Research Translator.

The repository should be reviewed as research software for governed research-to-decision translation. The core question is whether the repository makes the path from source material to decision artifact inspectable.

---

## Review Claim

The v1.1 release claims that applied AI research can be converted into governed decision artifacts through a controlled artifact chain:

```text
source material
  ↓
claims
  ↓
tasks
  ↓
evaluation plan
  ↓
run or review artifacts
  ↓
human gate
  ↓
decision summary
```

The claim is bounded. The release does not claim production readiness, compliance certification, legal sufficiency, or broad empirical validation.

## Strongest Evidence of Governance

The strongest evidence is not that the repository can produce an output. The strongest evidence is that it can stop.

Look at:

| Evidence | Why It Matters |
|---|---|
| `multi_agent_failure_modes_e0228882` | Demonstrates negative translation or rejection |
| `LIMITATIONS.md` | Prevents unsupported claims |
| `abstention-model.md` | Defines stopping conditions |
| `human-gate.md` | Preserves human authority |
| `audit-reconstruction.md` | Defines how a decision can be challenged later |

A governance system that always approves is an adoption workflow. This repository is stronger because rejection is part of the method.

## Standards Mapping Boundary

v1.1 includes mappings to NIST AI RMF, EU AI Act evidence concerns, and ISO/IEC 42001 management-system evidence.

These mappings should be read as interpretive support for review. They do not establish certification, conformity, legal classification, regulatory compliance, or enterprise-wide AI risk management.

| Mapping | Appropriate Use |
|---|---|
| NIST AI RMF | Explain how artifacts support Govern, Map, Measure, and Manage evidence |
| EU AI Act | Identify evidence relevant to documentation, oversight, record-keeping, and risk review |
| ISO/IEC 42001 | Show how the repository could support a process within an AI management system |

## Reviewer Decision Questions

A reviewer should be able to answer:

- Does the repository separate research usefulness from operational authorization?
- Does the method preserve source provenance?
- Are claims extracted before tasks are designed?
- Are tasks bounded before execution?
- Are evaluation criteria declared before output judgment?
- Can the system abstain or reject?
- Does human authority remain outside the model?
- Can a later reviewer reconstruct the decision?
- Do the limitations prevent overclaiming?

The release is credible only if these questions can be answered from artifacts.

## Suggested Reviewer Finding

Use this wording if the review supports the release:

```text
The v1.1 release strengthens Applied AI Research Translator as a research software artifact by documenting the governance method, artifact model, traceability chain, limitation register, and reviewer-facing controls required to inspect how applied AI research becomes a decision record. The release is appropriate for scholarly citation as software, with the boundary that it does not claim production readiness, regulatory compliance, certification, or general empirical validation.
```

Use this wording if the review identifies gaps:

```text
The v1.1 release provides a coherent governance and documentation layer, but the following artifacts require revision before the repository should be described as decision-complete for the reviewed use case: [list artifacts]. The release remains useful as research software if those limits are disclosed.
```

---

## Final Review Standard

The release passes review when the repository makes disagreement precise.

A reviewer should be able to say where they disagree: source, claim, task, evaluation, output, human gate, decision summary, limitation, or standards mapping.
