# v1.1 Archival Statement

Version 1.1 is an archival release of Applied AI Research Translator prepared for scholarly citation, Zenodo DOI registration, ORCID work listing, and institutional review.

The release is not defined by a new model capability. It is defined by the documentation and metadata required to make the repository inspectable as research software.

---

## Archival Claim

Applied AI Research Translator should be archived as software because it provides an implemented artifact system for governed research-to-decision translation.

It includes:

| Artifact Layer | Evidence |
|---|---|
| Source-to-claim method | `TRANSLATION-METHOD.md`, `claims.json` |
| Governance architecture | `GOVERNANCE-MODEL.md` |
| Traceability model | `TRACEABILITY.md` |
| Limitation register | `LIMITATIONS.md` |

The release is citable because the software and documentation together define a reproducible governance method.

## Why v1.1 Is More Than Documentation Cleanup

Documentation can be cosmetic. In this repository, documentation is part of the method.

The new v1.1 documents define the controls that allow a reviewer to inspect the path from research source to decision artifact. They specify the rationale, method, governance boundary, traceability chain, limitations, standards mappings, security considerations, and institutional review process.

| v1.1 Addition | Why It Is Methodologically Material |
|---|---|
| Research rationale | Defines the governance problem the repository addresses |
| Translation method | Defines the procedure by which research becomes claims and tasks |
| Governance model | Defines authority, abstention, human gate, and audit logic |
| Traceability model | Defines reconstructability as a property of the artifact chain |
| Limitation register | Prevents the release from overstating validity or compliance |
| Specifications | Defines artifacts, verdicts, abstention, human gate, and audit reconstruction |
| Governance documentation | Links the method to institutional review and standards-facing evidence |
| Citation metadata | Makes the release machine-readable for scholarly infrastructure |

The release changes how the repository can be reviewed, cited, and reused.

## Scholarly Contribution

The repository contributes a software-supported governance pattern: research claims should be translated into bounded decision artifacts before they influence operational action.

That contribution matters because applied AI research often enters practice through informal pathways: a paper is persuasive, a benchmark is impressive, a vendor report appears credible, or an online publication seems timely. The repository inserts a controlled artifact chain between source and decision.

```text
research usefulness
  ≠
operational authorization
```

The release operationalizes that distinction.

## Citation Boundary

The release can be cited for:

| Appropriate Citation Use | Explanation |
|---|---|
| Governed research translation | The repository defines and implements source-to-decision artifacts |
| Decision accountability architecture | Human gate and decision summaries preserve authority boundaries |
| AI governance research software | The repository provides schemas, scripts, packs, and runloop logic |
| Audit-ready decision records | The repository records source, claim, task, evaluation, output, and decision |
| Negative translation | The repository preserves rejection as a valid outcome |

The release should not be cited as evidence of:

| Unsupported Citation Use | Reason |
|---|---|
| Production safety | The repository is research software |
| Regulatory compliance | Legal obligations require independent review |
| General empirical validation | The pack corpus is demonstrative |
| Full agent governance | The model is intentionally bounded and non-agentic |
| Enterprise AI management system coverage | The repository supports a process, not a whole management system |

v1.1 archives the repository as research software for governed research-to-decision translation; it does not claim production deployment readiness, regulatory compliance, certification, or general empirical validation.
