# v1.0 Release Notes

Version 1.0 established Applied AI Research Translator as a decision-complete reference implementation for governed research-to-decision translation.

The release demonstrated that applied AI research can be translated into structured artifacts without allowing model output to become institutional authority by default.

## What v1.0 Established

v1.0 created the repository’s operational core.

| Area | v1.0 Contribution |
|---|---|
| Research packs | Introduced pack structure for translating research sources into claims, tasks, evaluation plans, and decision summaries |
| Schemas | Defined JSON Schema contracts for core governed artifacts |
| Runloop | Implemented a bounded execution path for controlled AI-assisted tasks |
| Human gate | Preserved human accept, override, and reject authority |
| Decision summaries | Created final audit-facing artifacts for completed runs or translation decisions |
| Negative translation | Demonstrated that rejection can be a valid governance outcome |
| Validation scripts | Added artifact validation utilities |
| Demo runs | Added locked example runs for review and reconstruction |

---

## Core Artifact Chain

```text
source material
  ↓
claims.json
  ↓
tasks.json
  ↓
eval_plan.json
  ↓
run artifacts
  ↓
human gate
  ↓
decision_summary.json
```

The release’s main design decision was to keep AI-generated content inside the evidence layer. Final decision authority remained outside the model.

---

## Governance Model in v1.0

| Governance Property | v1.0 Mechanism |
|---|---|
| Provenance | Source text stored inside packs |
| Claim discipline | Claims separated from papers and reports |
| Task containment | Tasks bounded before execution |
| Schema control | JSON Schemas required structured artifacts |
| Human authority | Human gate required accept, override, or reject |
| Audit reconstruction | Decision summaries preserved rationale and confidence |
| Rejection | Negative translation pack recorded stop logic |

v1.0 made the repository more than a prompt workflow. It made the source-to-decision path visible.

## v1.0 Limits

| Limit | Explanation |
|---|---|
| Limited documentation layer | The repository had operational artifacts but less research-facing explanation |
| Limited standards mapping | NIST AI RMF, EU AI Act, and ISO/IEC 42001 connections were not yet documented |
| Limited release metadata | Citation and archival material needed strengthening |
| Limited institutional review support | Review templates and governance controls needed clearer packaging |
| Limited pack corpus | The pack set demonstrated method, not broad empirical coverage |

These limits became the reason for v1.1.

---

## Appropriate Citation Interpretation

v1.0 should be cited as an initial research software implementation of a governed research-to-decision translation system. It should not be cited as proof that the method has been validated across domains, organizations, or deployment contexts.

Recommended wording:

```text
Version 1.0 established the reference implementation and core artifact chain for governed research-to-decision translation.
```

---

## Release Boundary

v1.0 did not provide production deployment readiness, legal compliance, regulatory certification, enterprise AI inventory management, continuous monitoring, or full AI management system coverage.

It provided a controlled reference implementation for making research translation inspectable.
