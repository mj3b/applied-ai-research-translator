# HAIC Reliance Review Pack

This pack translates human-AI collaboration research into reliance-calibration review artifacts. The pack asks whether a human decision process can use AI assistance without turning the model output into an unexamined decision.

The decision summary classifies the source as suitable for task design with medium confidence. The reason is bounded: reliance behavior can be observed through user interaction with AI recommendations, override behavior, confidence presentation, and review records.

## What this pack evaluates

The pack focuses on reliance quality. It looks for the conditions under which a human reviewer appropriately accepts, rejects, questions, or escalates an AI recommendation. The governance problem is straightforward: an AI system can appear useful while shifting judgment away from the human decision owner.

This pack turns that problem into reviewable artifacts:

- claims about human-AI complementarity and reliance calibration
- tasks that evaluate over-reliance, under-reliance, override quality, and review completeness
- an evaluation plan for measuring whether AI assistance improves decisions without eroding human authority
- a decision summary that records why the source may proceed into bounded task design

## Appropriate use

Use this pack for decision-support settings where a human reviewer remains responsible for the final decision. Good candidates include triage, prioritization, case review, quality review, recommendation review, or workflow assistance.

Avoid using this pack for autonomous action, silent automation, or workflows where the human reviewer cannot inspect the evidence behind the AI recommendation.

## Governance boundary

This pack supports task design. It does not certify production deployment. Before operational use, the reviewer should refine line-referenced citations, add a challenge set, and define acceptance thresholds for reliance calibration.
